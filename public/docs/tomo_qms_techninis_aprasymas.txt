Tomo-QMS — Pilnas techninis sistemos aprašymas
================================================================================

1. APŽVALGA
================================================================================
Tomo-QMS (Quality Management System) yra žiniatinklio pagrindu veikianti gamybos
kokybės valdymo sistema, sukurta UAB „ELGA" gamybos procesų kokybės kontrolei.
Sistema apima užsakymų valdymą, funkcinių ir dielektrinių bandymų registravimą,
komponentų sekimą, gaminio paso generavimą, pretenzijų valdymą, matavimo
prietaisų priežiūrą ir kokybės statistiką.

Technologijos: PHP 8.3 / PostgreSQL (PDO) / mPDF / Resend API
Serveris: PHP integruotas serveris (php -S 0.0.0.0:5000)
Produkcinė sistema: https://nkokybe.elga.tech
El. pašto siuntėjas: Tomo-QMS <noreply@updates.elga.tech>

2. TECHNINĖ ARCHITEKTŪRA
================================================================================

2.1 Serverio paleidimas (server/index.ts)
- Node.js procesas paleidžia PHP integruotą serverį komanda:
  php -S 0.0.0.0:5000 -t public public/router.php
- Automatinis perkrovimas: jei PHP procesas sustoja, palaukiama 1-2 sek. ir
  paleidžiamas iš naujo
- Startuojant valomi pasenusios PHP sesijos: rm -f /tmp/sess_*

2.2 URL maršrutizavimas (public/router.php)
- Šakninis / nukreipia į login.php
- Švarios URL: /uzsakymai -> uzsakymai.php (automatiškai pridedamas .php)
- Statiniai failai (CSS, JS, paveikslėliai) grąžinami tiesiogiai
- Sveikatos tikrinimas: /__healthcheck ir /__replit_health grąžina "OK"
- Nežinomoms URL grąžinamas 404 puslapis

2.3 Aplinkos kintamieji
- DATABASE_URL — vietinės PostgreSQL DB prisijungimas
- TOMO_QMS_DATABASE_URL — nuotolinės TomoQMS DB (eksportui)
- QUALITY_TOMAS_DATABASE_URL — quality_tomas DB (importui)
- RESEND_API_KEY — Resend.com API raktas el. laiškų siuntimui

5. AUTENTIKACIJA IR SAUGUMAS
================================================================================

5.1 Prisijungimas (login.php)
- Priima vardą, pavardę ir slaptažodį
- Ieško vartotojo pagal vardą ir pavardę vartotojai lentelėje
- Tikrina slaptažodį su password_verify() (bcrypt — PASSWORD_BCRYPT)
- Sėkmės atveju: session_regenerate_id(true) prieš sesijos kintamųjų nustatymą
- Nustato: vartotojas_id, vardas, pavarde, role, paskutine_veikla
- Registruoja aktyvų vartotoją aktyvus_vartotojai lentelėje

5.2 Sesijos konfigūracija (Sesija.php)
- Galiojimas: 1800 sekundžių (30 minučių)
- Slapuko parametrai:
  - lifetime: 0 (iki naršyklės uždarymo)
  - secure: true (tik HTTPS)
  - httponly: true (neprieinamas JavaScript)
  - samesite: Lax
- Kiekviena užklausa atnaujina $_SESSION['paskutine_veikla']
- Atsitiktinai (1 iš 10) valo neaktyvius vartotojus iš aktyvus_vartotojai

5.3 "Prisiminti mane" mechanizmas
- Generuojamas žetonas: bin2hex(random_bytes(32)) — 64 simbolių
- Naršyklėje: remember_token slapukas
- DB: SHA-256 maišos reikšmė remember_tokens lentelėje
- Galiojimas: 30 dienų
- Atsijungiant: trinamas iš DB ir naršyklės

7. MODULIŲ APRAŠYMAI
================================================================================

7.1 Gamybos modulių valdymas (moduliai.php)
- Rodomi visi gamybos moduliai (gaminiu_rusys lentelė)
- Admin gali kurti naujus modulius
- Dublikatų prevencija: tikrinama ar pavadinimas jau egzistuoja prieš sukuriant
- Pasirinktas modulis saugomas sesijoje (aktyvus_grupe, aktyvus_modulis_pav)

7.2 Užsakymų valdymas (uzsakymai.php)
Sukūrimas:
- Įvedamas užsakymo numeris, kiekis, užsakovas, objektas
- Automatiškai generuojami gaminiai pagal kiekį
  (pvz. kiekis=3 → gaminiai: 16467-1, 16467-2, 16467-3)
- Galima nustatyti per-order įmonės duomenis (PDF generavimui)

Sąrašo vaizdas:
- Filtruojama pagal aktyvią gaminių rūšį (gaminiu_rusis_id)
- Rikiuojama: sukurtas DESC, id DESC
- Rodomas PDF būklės indikatorius kiekvienam gaminiui
  (paso PDF, dielektrinių PDF, funkcinių PDF)

Detali peržiūra (?id=X):
- Visi užsakymo gaminiai su mygtukais bandymams
- Galimybė pervardyti individualius gaminius

Trynimas:
- Tik admin rolei
- Reikia patvirtinti įvedant tikslų užsakymo numerį
- Kaskadinis trynimas: funkciniai_bandymai, komponentai,
  dielektriniai_bandymai, saugikliu_ideklai, izeminimo_tikrinimas,
  paso_teksto_korekcijos, pretenzijos (su nuotraukomis), gaminiai

- 21 gamybos reikalavimas (arba pagal šabloną iš funkciniu_sablonas)

- Aukštos įtampos elektros saugos testai
- Sekcijos:
  a) Bandymų prietaisai (CRUD) — modelis, numeris, kalibravimo data
  b) Vidutinės įtampos bandymai (6-24 kV) — kabelių testavimas
  c) Žemos įtampos bandymai (0.4 kV) — izoliacijos varža
  d) Antrinių grandinių bandymai
  e) Įžeminimo grandinių tikrinimas — varžos matavimas (Ω)

- Duomenys: dielektriniai_bandymai, antriniu_grandiniu_bandymai,

7.5 Sumontuoti komponentai (MT/mt_sumontuoti_komponentai.php)

7.6 Gaminio pasas (MT/mt_pasas.php)

7.7 Pretenzijų valdymas (pretenzijos.php)
Gyvavimo ciklas:
  nauja → tyrimas → vykdoma → uzbaigta
                              → atmesta
  Kai statusas = uzbaigta|atmesta → auto uzbaigimo_data = CURRENT_DATE

Sukūrimas:
- Privalomas: aprasymas
- Tipas: vidine/kliento/tiekejo
- Galima susieti su užsakymu (dropdown) arba įvesti rankinį numerį
- Galima įkelti nuotraukas (multiple) → BYTEA pretenzijos_nuotraukos
- Galima įkelti defekto PDF → BYTEA pretenzijos.defekto_pdf_turinys

Filtravimas:
- GET ?tipas=vidine|kliento|tiekejo
- GET ?statusas=nauja|tyrimas|vykdoma|uzbaigta|atmesta
- Rikiavimas: statusas prioritetas (nauja=1, tyrimas=2...), tada sukurta DESC

Statistikos kortelės:
- Viso pretenzijų, naujų, aktyvių (nauja+tyrimas+vykdoma), užbaigtų

Viešoji peržiūra:

10. SINCHRONIZACIJOS SISTEMA
================================================================================

10.1 Architektūra
Sistema naudoja dvi nuotolines duomenų bazes:
- TOMO_QMS_DATABASE_URL — eksporto tikslas (test rezultatų siuntimas)
- QUALITY_TOMAS_DATABASE_URL — importo šaltinis (užsakymų, gaminių gavimas)

10.2 Importas (quality_tomas → vietinė DB)
Paleidimas: sinchronizuoti.php POST importas=1
Metodas: TomoQMS::importuotiILocalDB()

Fazės:
1. Užsakymai — filtruoja pagal gaminiu_rusis_id=2 (MT),
   sinchronizuoja uzsakovai ir objektai
2. Gaminiai ir bandymai — iteruoja per importuotus užsakymus,
   traukia gaminius, funkcinius bandymus (mt_funkciniai_bandymai),
   komponentus (mt_komponentai)
3. Pretenzijos — traukia pretenzijas su nuotraukomis ir el. pašto istorija,
   naudoja qt_pretenzija_id stabiliam identifikavimui,
   mapuoja QT gaminys_id → vietinį gaminio_id

10.3 Eksportas (vietinė DB → TomoQMS)
Paleidimas: sinchronizuoti.php POST masinis=1 arba uzsakymo_id=X

Sinchronizuojami duomenys (per sinchronizuotiGamini() funkciją):
- sinchFunkciniai — funkciniai bandymai su defektų nuotraukomis
- sinchKomponentai — komponentų sąrašai
- sinchDielektriniai — dielektriniai, antrinių grandinių, įžeminimo bandymai
- sinchSaugiklius — saugiklių įdėklai
- sinchPrietaisus — bandymų prietaisai
- sinchProtokoloNr — protokolų numeriai
- sinchPasoTeksta — paso teksto korekcijos
- sinchPDF — sugeneruoti PDF failai

Eksporto logika: DELETE esami + INSERT nauji (pilnas perrašymas)

10.4 Sinchronizacijos žurnalas (sync_log)
- Saugomas nuotolinėje TomoQMS DB
- Kiekviena operacija registruojama su: data, veiksmas, lentelė,
  užsakymo numeris, įrašų kiekis, statusas (ok/klaida), klaidos tekstas
- Peržiūra: sync_log.php
